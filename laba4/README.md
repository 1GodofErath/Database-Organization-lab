# Database-Organization-lab
# Лабораторна робота 4: Система управління базою даних

## Призначення

Веб-додаток для управління базою даних з функціоналом автентифікації користувачів та виконання CRUD операцій.  Проект реалізований на PHP з використанням MySQL та Docker для контейнеризації.

## Структура проекту

```
laba4/
├── . env                    # Змінні оточення (конфігурація БД)
├── .gitattributes         # Атрибути Git
├── Dockerfile             # Образ Docker для PHP
├── Dockerfile.mysql       # Образ Docker для MySQL
├── Dockerfile. php         # Образ Docker для PHP (альтернативний)
├── docker-compose.yml     # Конфігурація Docker Compose
├── init.sql               # SQL скрипт ініціалізації БД
├── db.php                 # Конфігурація підключення до БД
├── index.php              # Головна сторінка з CRUD операціями
├── login.php              # Сторінка входу
├── register.php           # Сторінка реєстрації
├── logout.php             # Скрипт виходу
├── mysql_keyring.cnf      # Конфігурація MySQL keyring
├── style/                 # CSS стилі
└── dah/                   # Додаткові ресурси
```

## Інструкція запуску

### Передумови
- Docker
- Docker Compose
- Git

### Крок 1: Клонування репозиторію
```bash
git clone https://github.com/1GodofErath/Database-Organization-lab.git
cd Database-Organization-lab/laba4
```

### Крок 2: Налаштування змінних оточення
Перевірте файл `.env` та налаштуйте параметри за потреби:
```env
MYSQL_ROOT_PASSWORD=your_password
MYSQL_DATABASE=your_database
MYSQL_USER=your_user
MYSQL_PASSWORD=your_password
```

### Крок 3: Запуск Docker контейнерів
```bash
# Перейдіть до кореневої директорії репозиторію
cd Database-Organization-lab

# Додайте README.md (якщо ще не додано)
git add laba4/README.md
git commit -m "docs: додано детальний README.md для лабораторної роботи 4"
git push origin main

# Створіть тег для лабораторної роботи 4
git tag -a lab4-v1.0. 0 -m "Лабораторна робота 4: Система управління БД з CRUD операціями"

# Відправте тег на GitHub
git push origin lab4-v1.0.0


docker-compose up -d
```

### Крок 4: Доступ до додатку
Відкрийте браузер та перейдіть за адресою:
```
http://localhost:8080
```

### Зупинка додатку
```bash
docker-compose down
```

## Як тестувати

### 1. Тестування реєстрації
- Перейдіть на сторінку реєстрації
- Введіть ім'я користувача та пароль
- Натисніть "Зареєструватися"
- Перевірте успішне створення облікового запису

### 2. Тестування входу
- Використайте зареєстровані облікові дані
- Перейдіть на сторінку входу (`login.php`)
- Введіть дані та увійдіть до системи

### 3.  Тестування CRUD операцій
Після успішного входу ви маєте доступ до операцій:

#### Create (Створення)
- Заповніть форму для додавання нового запису
- Натисніть "Додати"
- Перевірте, що запис з'явився у списку

#### Read (Читання)
- Перегляньте список всіх записів на головній сторінці
- Перевірте коректність відображення даних

#### Update (Оновлення)
- Виберіть запис для редагування
- Змініть дані у формі
- Збережіть зміни
- Перевірте оновлення даних

#### Delete (Видалення)
- Виберіть запис для видалення
- Підтвердіть видалення
- Перевірте, що запис більше не відображається

### 4. Тестування безпеки
- Спробуйте отримати доступ до `index.php` без входу (має перенаправити на `login.php`)
- Перевірте хешування паролів у базі даних
- Тестуйте вихід з системи (`logout.php`)

##  Сутності та CRUD операції

### Сутність: Users (Користувачі)
**Таблиця:** `users`

**Поля:**
- `id` (INT, PRIMARY KEY, AUTO_INCREMENT) - Унікальний ідентифікатор
- `username` (VARCHAR) - Ім'я користувача
- `password` (VARCHAR) - Хешований пароль
- `created_at` (TIMESTAMP) - Дата створення

**CRUD операції:**
- ✅ **Create**: Реєстрація нового користувача через `register.php`
- ✅ **Read**: Автентифікація через `login.php`
- ❌ **Update**: Не реалізовано в поточній версії
- ❌ **Delete**: Не реалізовано в поточній версії

##  Технології

- **Backend**: PHP 8.5+
- **Database**: MySQL 8.3
- **Containerization**: Docker & Docker Compose
- **Frontend**: HTML5, CSS3
- **Security**: Password hashing, Session management

##  Примітки

- База даних автоматично ініціалізується при першому запуску через `init.sql`
- Паролі зберігаються у хешованому вигляді
- Сесії використовуються для управління автентифікацією
- Додаток працює на порту 8080 (можна змінити в `docker-compose.yml`)
